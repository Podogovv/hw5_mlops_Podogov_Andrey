Ограничения ноутбуков по сравнению с Marimo:

Google Colab удобен для быстрых экспериментов, но их модель выполнения остаётся императивной. 

Ячейки можно запускать в любом порядке, из-за чего состояние становится неявным и легко ломается. Это напрямую влияет на воспроизводимость. Результат в Colab часто зависит от истории выполнения, а не только от кода. В Marimo зависимости между ячейками явные, поэтому вычисления полностью определены.

Colab плохо подходит для масштабирования проектов. По мере роста кода становится трудно поддерживать структуру, переиспользование и перенос в production. Marimo ближе к обычному коду и лучше вписывается в инженерные практики.

С контролем версий тоже есть проблемы. Формат .ipynb содержит лишние данные и плохо читается в diff. У Marimo текстовое представление чище, поэтому удобнее работать с git.

Google Colab ограничен средой: сессии временные, ресурсы нестабильны. Marimo чаще используется в контролируемом окружении, что делает его более надёжным для разработки ML-систем.

Шаги для воспроизведения:

1. Клонирование репозитория
&#x20; git clone <repo-url>
&#x20; cd mlops\_project

2. Установка зависимостей:
&#x20;pip install -r requirements.txt

3. Получение данных:
&#x20;dvc pull

4. Запуск пайплайна:
&#x20;dvc repro

5. Просмотр метрик в MLflow:
&#x20;mlflow ui --backend-store-uri sqlite:///mlflow.db
