## Ограничения ноутбуков по сравнению с Marimo:

Google Colab удобен для быстрых экспериментов, но их модель выполнения остаётся императивной, код выполняется строка за строкой. 

Ячейки можно запускать в любом порядке, из-за чего состояние становится неявным и легко ломается. Это напрямую влияет на воспроизводимость. Результат в Colab часто зависит от истории выполнения, а не только от кода. В Marimo зависимости между ячейками явные, поэтому вычисления полностью определены.

Colab плохо подходит для масштабирования проектов. По мере роста кода становится трудно поддерживать структуру, переиспользование и перенос в production. Marimo ближе к обычному коду и лучше вписывается в инженерные практики.

С контролем версий тоже есть проблемы. Формат .ipynb содержит лишние данные и плохо читается в diff. У Marimo текстовое представление чище, поэтому удобнее работать с git.

Google Colab ограничен средой: сессии временные, ресурсы нестабильны. Marimo чаще используется в контролируемом окружении, что делает его более надёжным для разработки ML-систем.

## Описание
Проект предсказания выживаемости пассажиров Титаника с использованием DVC для управления данными и пайплайном, и MLflow для трекинга экспериментов.
________________________________________________
## Особенности
- Полная воспроизводимость экспериментов
- Автоматическая обработка пропущенных значений
- Feature engineering (FamilySize, IsAlone)
- Сравнение моделей: RandomForest, GradientBoosting, LogisticRegression
- Визуализация confusion matrix и feature importance
________________________________________________
## Датасет
Titanic dataset содержит информацию о пассажирах:
- 891 запись
- 12 признаков (Pclass, Sex, Age, SibSp, Parch, Fare, Embarked и др.)
- Целевая переменная: Survived (0 - погиб, 1 - выжил)

## Краткое описание пайплайна
**Стадия 1: Подготовка данных (prepare)**

\- Удаление неинформативных признаков 

\- Заполнение пропусков в возрасте 

\- Заполнение пропусков в порте посадки модой

\- Создание новых признаков

\- Кодирование категориальных признаков 

\- Стандартизация числовых признаков (Age, Fare, SibSp, Parch, FamilySize)

\- Разделение на train/test


**Стадия 2: Обучение модели (train)**

\- Загрузка гиперпараметров из params.yaml

\- Обучение модели 

\- Расчет метрик

\- Логирование всего в MLflow 

\- Сохранение модели в model.pkl

## Шаги для воспроизведения:

**1. Клонирование репозитория**

git clone hw5_mlops_Podogov_Andrey hw5_mlops_Podogov_Andrey_test

cd hw5_mlops_Podogov_Andrey_test

**2. Установка зависимостей**

pip install -r requirements.txt

**3. Получение данных**

dvc pull

**4. Запуск пайплайна**

dvc repro

**5. Запуск MLflow UI для просмотра результатов**

mlflow ui --backend-store-uri sqlite:///mlflow.db

**6. Просмотр метрик**

cat metrics.json
